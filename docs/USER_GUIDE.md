# 庫存管理系統 (Inventory Management System) - 用戶操作指南

歡迎使用本**庫存管理系統**。本指南將協助您建立編譯環境、建置專案、執行程式，並詳細說明各項功能的操作步驟與業務流程。

---

## 1. 環境準備與編譯指南

本系統採用標準 C++17 撰寫，使用 CMake 作為建置工具。以下說明在 Windows 環境下使用 Visual Studio 的編譯步驟：

### 1.1 前置需求
* **作業系統**：Windows 10 / 11
* **編譯器**：Visual Studio 2022 或更新版本（包含 C++ 開發套件，支援 MSVC 19.30+）
* **建置工具**：CMake (3.10 或更新版本)
  * *註：Visual Studio 已內建 CMake，可直接使用 VS 提供的 CMake 執行檔。*

### 1.2 編譯步驟
開啟 PowerShell 或 命令提示字元 (CMD)，執行以下指令：

1. **進入專案目錄**：
   ```powershell
   cd C:\Users\bolin\Inventory-Management-System
   ```
2. **配置 CMake 專案**（產生 Visual Studio 建置檔）：
   使用 Visual Studio 內建的 CMake 進行配置：
   ```powershell
   & "C:\Program Files\Microsoft Visual Studio\18\Community\Common7\IDE\CommonExtensions\Microsoft\CMake\CMake\bin\cmake.exe" -B build -S .
   ```
3. **編譯專案**：
   進行 Debug 版本的編譯建置：
   ```powershell
   & "C:\Program Files\Microsoft Visual Studio\18\Community\Common7\IDE\CommonExtensions\Microsoft\CMake\CMake\bin\cmake.exe" --build build --config Debug
   ```
4. **編譯產物**：
   編譯成功後，執行檔將生成於：
   `C:\Users\bolin\Inventory-Management-System\build\Debug\inventory_system.exe`

---

## 2. 啟動與操作流程

### 2.1 啟動程式
在終端機中執行：
```powershell
.\build\Debug\inventory_system.exe
```
啟動後，系統會自動檢查並加載 `data/` 目錄下的歷史商品與交易記錄。

---

## 3. 功能操作手冊

系統採用樹狀選單設計，輸入數字並按 `Enter` 即可進入子選單或執行功能。在大部分輸入畫面中，您可以隨時輸入 `0` 或按規定格式取消操作。

### 3.1 主選單 (Main Menu)
系統主選單包含四大功能區塊與退出選項：
```
=== 庫存管理系統 ===
1. 商品管理
2. 庫存操作
3. 報表查詢
4. 數據管理
0. 退出
```

---

### 3.2 商品管理選單 (Product Management)
進入主選單 `1`，可對商品的「品項定義」進行維護。

#### 1. 新增商品 (Add Product)
* **流程**：
  1. 選擇商品類型：`1. 普通商品`、`2. 易腐商品`、`3. 電子產品`。
  2. 系統自動分配新 ID（如 `1001`）。
  3. 輸入名稱、價格（如 `19.99`）、初始庫存、供應商。
  4. 根據類型輸入特有欄位：
     * 普通商品：輸入商品類別（如 `食品`、`文具`）。
     * 易腐商品：輸入過期日期，格式必須為 `YYYY-MM-DD`（如 `2026-12-31`）。
     * 電子產品：輸入保固期月數（如 `24`）。
  5. 顯示 `✓ 商品新增成功！`。

#### 2. 查詢商品 (View Product)
* 輸入商品 ID，系統將呈現該商品的當前狀態與特有屬性（如過期天數或折舊後單價）。

#### 3. 修改商品 (Modify Product)
* 系統顯示商品清單，輸入欲修改的商品 ID。
* 選擇要修改的欄位：`1. 修改名稱`、`2. 修改價格`、`3. 修改供應商`。
* 輸入新值後即完成更新。

#### 4. 刪除商品 (Delete Product)
* 系統顯示商品清單，輸入商品 ID。
* 系統會展示該商品詳情，並提示 `確認刪除? (yes/no)`。
* 輸入 `yes` 確認刪除，該商品將被移出系統。

#### 5. 列表所有商品 (List All Products)
* 列出目前內存中的所有商品。

---

### 3.3 庫存操作選單 (Inventory Operations)
進入主選單 `2`，可對現有商品進行進出貨實體操作。

#### 1. 進貨 (Stock In)
* 系統在畫面上展示所有商品的 ID 與名稱。
* 輸入商品 ID（輸入 `0` 可取消返回）。
* 輸入「進貨數量」與「進貨單價」。
* 進貨成功後，庫存相應增加，並記入交易歷史。

#### 2. 出貨 (Stock Out)
* 系統展示所有商品清單。
* 輸入商品 ID（輸入 `0` 可取消返回）。
* 輸入「出貨數量」與「出貨單價」。
* **出貨規則約束**：
  * 若出貨數量大於當前庫存，提示 `✗ 出貨失敗或庫存不足！`。
  * 若該商品為易腐商品且**已過期**，系統將強制拒絕出貨並提示 `Cannot remove expired product!`。

#### 3. 低庫存警告 (Low Stock Alert)
* 顯示所有庫存數量 $\le 10$ 的商品，提醒管理人員及時進行進貨採購。

#### 4. 交易紀錄 (Transaction History)
* 依時間序列展示所有的進出貨歷史紀錄，包含每次交易的精確時間戳記、單價與總金額。

---

### 3.4 報表查詢選單 (Reports & Statistics)
進入主選單 `3`，可檢視倉庫的財務與統計報表。

#### 1. 庫存統計報表
* 彙總統計：商品總種類數、商品總件數、庫存總價值，並逐一列出各商品庫存數量。

#### 2. 價值統計報表
* 系統自動計算每種商品的總價值（電子產品會自動折算折舊殘值，過期食品價值自動歸零）。
* 依**總價值由高到低（降冪）**排序輸出，幫助掌握核心資產分布。

#### 3. 商品類型報表
* 分類統計「普通商品」、「易腐商品」與「電子產品」的種類件數與各自的資產價值。

#### 4. 交易報表
* 統計系統歷史進貨總支出與出貨總收入，便於評估資金流水與利潤。

---

### 3.5 數據管理選單 (Data Management)
進入主選單 `4`，可手動進行數據備份與回復。
* **保存數據**：將當前內存的商品與交易資料立即寫入 `data/products.csv` 與 `data/transactions.csv`。
* **加載數據**：清空當前內存，從 CSV 檔案重新載入數據。
* *註：程式啟動時會自動加載，正常退出時會自動保存，日常操作無需頻繁手動保存。*

## 4. 業務場景實操範例 (Walkthrough Example)

以下為一個完整的模擬操作流程，供您快速熟悉系統：

### 場景：引入並管理一批易腐鮮奶

1. **步驟 1：定義新商品**
   * 進入選單 `1. 商品管理` -> `1. 新增商品`。
   * 選擇 `2. 易腐商品(食品等)`。
   * 輸入名稱：`瑞穗鮮乳`
   * 輸入價格：`3.50`
   * 輸入初始庫存：`0`（先建立品項，庫存為 0）
   * 輸入供應商：`統一企業`
   * 輸入過期日期：`2026-06-30` (假設今天為 2026-06-24，尚餘 6 天過期)
   * 系統提示：`✓ 商品新增成功！(ID: 1001)`

2. **步驟 2：執行進貨**
   * 返回主選單，進入 `2. 庫存操作` -> `1. 進貨`。
   * 系統顯示商品清單，看見 `1001 | 瑞穗鮮乳`。
   * 輸入商品 ID：`1001`。
   * 輸入進貨數量：`50`
   * 輸入進貨單價：`3.00` (採購價)
   * 系統提示：`✓ 進貨成功！`。此時瑞穗鮮乳庫存變為 50 件。

3. **步驟 3：查看價值報表**
   * 返回主選單，進入 `3. 報表查詢` -> `2. 價值統計報表`。
   * 系統顯示瑞穗鮮乳價值為 `$175.00`（$3.50 \times 50$），此時狀態顯示為 `(6 days left)`。

4. **步驟 4：模擬過期與出貨受阻（業務安全驗證）**
   * 假設時間流逝，鮮乳已過期（有效期限已過）。
   * 再次進入 `3. 報表查詢` -> `2. 價值統計報表`。
     * 系統偵測到目前日期已大於 `2026-06-30`，鮮乳狀態變為 `[EXPIRED]`。
     * 瑞穗鮮乳的庫存價值在報表中**自動歸零 (`$0.00`)**。
   * 返回主選單，進入 `2. 庫存操作` -> `2. 出貨`。
     * 輸入商品 ID：`1001`。
     * 系統印出：`Cannot remove expired product!` 並提示 `✗ 出貨失敗或庫存不足！`。
     * **成功攔截過期食品出貨，保障業務合規與食品安全！**
